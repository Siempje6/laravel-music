
Onderzoek & Foutenanalyse Je had PHP 8.4 geïnstalleerd maar je terminal
gebruikte nog PHP 8.0. Daardoor bleef Laravel foutmeldingen geven zoals
“Your Composer dependencies require PHP >= 8.1, but you are running
8.0.30”. De reden is dat Homebrew PHP 8.4 installeert in
/usr/local/opt/php@8.4, maar macOS je PATH nog steeds naar PHP 8.0 laat
wijzen. De PHP download op php.net bevat geen macOS‑binaries, alleen
source code die je zelf zou moeten compileren—dus dat kun je niet
gebruiken.

Fouten uitleggen & Correctie Fout: PHP 8.4 was wél geïnstalleerd maar
niet actief. Je terminal keek nog steeds naar PHP 8.0. Correctie: Voeg
PHP 8.4 toe aan je PATH met:

export
PATH=“/usr/local/opt/php@8.4/bin:$PATH" export PATH="/usr/local/opt/php@8.4/sbin:$PATH”

Voer daarna uit: source ~/.zshrc

Nu zou php -v PHP 8.4.x moeten laten zien. Daarna kun je composer update
en php artisan serve uitvoeren en werkt alles normaal.

Bronnenlijst - Homebrew FAQ: https://docs.brew.sh/FAQ - PHP downloads:
https://www.php.net/downloads.php - Laravel Server Requirements:
https://laravel.com/docs/11.x/deployment#server-requirements - Composer
platform-check: https://getcomposer.org/doc/03-cli.md#platform-check

Korte samenvatting PHP 8.4 stond al geïnstalleerd maar je PATH wees nog
naar PHP 8.0. Door PHP 8.4 bovenaan je PATH te zetten werkt php -v én
Laravel weer goed.
